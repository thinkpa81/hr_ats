# 코아시아 ATS (Applicant Tracking System)

## 📋 프로젝트 개요

**차세대 지능형 채용관리 시스템** - 코아시아 지주사 및 전 계열사를 위한 통합 ATS 플랫폼

### 🎯 프로젝트 목표
- **표준화된 채용 프로세스**: 지주사 기준 표준 채용 워크플로우 확립
- **AI 기반 지능형 매칭**: 지원자-직무 적합도 자동 분석 및 추천
- **데이터 기반 의사결정**: 채용 퍼널 분석 및 실시간 통계 대시보드
- **전사 공용 시스템**: 6개 법인 통합 채용 데이터베이스 운영

### ✨ 주요 기능
- ✅ **실시간 대시보드**: 채용 현황, AI 매칭 점수, 단계별 지원자 통계
- ✅ **이력서 직접 업로드**: AI 기반 자동 분석 및 적합도 판단 (학력, 경력, 기술 스택 자동 추출)
- ✅ **지능형 지원자 관리**: AI 매칭점수 자동 계산, 서류 스크리닝, 상태 추적
- ✅ **이메일 접수 관리**: 발송 내역, 템플릿 관리, 상태 추적
- ✅ **면접 프로세스 관리**: 일정 관리, 평가 입력, 면접관 피드백 시스템
- ✅ **통계 분석**: 채용 퍼널 분석, 채용공고별 성과, 전환율 추적
- ✅ **프로세스 로그**: 모든 채용 활동 자동 기록 및 추적
- ✅ **이메일 알림 시스템**: 지원서 접수, 면접 안내, 합격 통보 자동화

## 🗄️ 데이터 아키텍처

### 데이터 모델
본 시스템은 **Cloudflare D1 (SQLite 기반)** 데이터베이스를 사용하여 다음 9개의 핵심 테이블로 구성됩니다:

#### 1️⃣ 채용공고 (job_postings)
- 법인별 채용공고 관리 (지주사, 계열사)
- 직무 상세, 자격요건, 급여범위, 복리후생
- 상태 추적 (open/closed/draft)

#### 2️⃣ 지원자 (applicants)
- 지원자 기본 정보 (연락처, 학력, 경력)
- 이력서, 포트폴리오, SNS 링크
- 지원 경로 추적

#### 3️⃣ 지원내역 (applications)
- 지원자-채용공고 매칭 정보
- **AI 매칭 점수** (0-100) 및 매칭 근거
- 서류전형 점수, 현재 단계, 상태
- 최종 합격/불합격 결정 이력

#### 4️⃣ 면접일정 (interviews)
- 면접 유형 (1차/2차/임원/PT)
- 일시, 장소, 면접 방식 (대면/화상/전화)
- 면접관 정보 (JSON 배열)

#### 5️⃣ 면접평가 (interview_evaluations)
- 5개 평가 항목별 점수 (기술역량, 의사소통, 조직적합성, 문제해결, 리더십)
- 강점/약점 분석
- 면접관 추천의견 (적극추천/추천/보류/비추천)

#### 6️⃣ 평가항목 마스터 (evaluation_criteria)
- 채용공고별 맞춤형 평가 기준
- 항목별 가중치 설정
- 평가 유형 (서류/면접/기술테스트)

#### 7️⃣ 프로세스 로그 (process_logs)
- 모든 채용 활동 자동 기록
- 단계별 액션 추적
- 담당자 및 시간 기록

#### 8️⃣ 이메일 로그 (email_logs)
- 지원자 커뮤니케이션 이력
- 이메일 유형별 템플릿
- 발송 상태 추적

#### 9️⃣ 통계 스냅샷 (statistics_snapshot)
- 일일 채용 통계 집계
- 단계별 지원자 분포
- 평균 채용 기간, 전환율

### 데이터 흐름
```
지원서 접수 → AI 매칭 분석 → 서류전형 → 면접 일정 → 면접 평가 → 최종 결정
     ↓              ↓              ↓           ↓           ↓           ↓
  [프로세스 로그 자동 기록] + [이메일 알림 발송] + [통계 데이터 갱신]
```

## 📊 API 엔드포인트

### 대시보드 API
- `GET /api/dashboard/stats` - 채용 현황 통계 (KPI 지표, 단계별 분포, 최근 지원자)

### 채용공고 API
- `GET /api/job-postings` - 채용공고 목록
- `GET /api/job-postings/:id` - 특정 채용공고 상세

### 지원자 관리 API
- `GET /api/applications` - 지원자 목록 (필터링, 정렬 지원)
- `GET /api/applications/:id` - 지원자 상세 (면접, 평가, 로그 포함)
- `PUT /api/applications/:id/status` - 지원자 상태 업데이트
- `POST /api/applications/:id/ai-match` - AI 매칭 점수 계산
- `POST /api/applicants/upload-resume` - **신규** 이력서 직접 업로드 및 AI 자동 분석

### 면접 관리 API
- `POST /api/interviews` - 면접 일정 생성
- `POST /api/interview-evaluations` - 면접 평가 저장

### 이메일 관리 API (신규)
- `GET /api/emails` - 이메일 발송 내역 조회 (상태별, 유형별 필터링)
- `POST /api/emails/send` - 이메일 발송 (템플릿 지원)
- `PUT /api/emails/:id/status` - 이메일 상태 업데이트

### 통계 분석 API
- `GET /api/analytics/funnel` - 채용 퍼널 분석
- `GET /api/analytics/by-job-posting` - 채용공고별 통계

## 🚀 사용자 가이드

### 1. 대시보드 확인
메인 페이지 접속 시 자동으로 **채용 현황 대시보드**가 표시됩니다:
- 진행중인 채용공고 수
- 총 지원자 수 및 지원 건수
- 오늘 면접 일정
- AI 매칭 평균 점수
- 단계별 지원자 현황
- 최근 지원자 목록 (상위 5명)

### 2. 이력서 직접 업로드 (신규 기능!)
**이력서 업로드** 탭에서 지원자 이력서를 직접 입력하고 AI가 자동 분석합니다:
- **채용공고 선택**: 지원할 직무 선택
- **기본 정보 입력**: 이름, 이메일, 연락처, 생년월일, 성별
- **이력서 내용 입력**: 학력, 경력, 기술 스택 등 자유 형식으로 작성
- **자기소개서 입력** (선택): 지원 동기 및 포부

**AI 자동 분석 항목:**
- 학력 자동 추출 (학사/석사/박사)
- 경력 연수 계산
- 기술 스택 키워드 추출
- 대학교 이름 인식
- **AI 매칭 점수 자동 계산** (100점 만점)
- 매칭 근거 자동 생성

제출 즉시 AI 분석 결과가 표시되며, 지원자 관리 탭에서 확인할 수 있습니다.

### 3. 지원자 관리
**지원자 관리** 탭에서 다음 작업을 수행할 수 있습니다:
- **필터링**: 상태(제출됨/서류전형/면접/제안/채용완료/불합격), 단계별 검색
- **정렬**: 지원일, AI 매칭점수, 서류점수 기준 정렬
- **상세 보기**: 각 지원자의 상세 정보, 면접 이력, 평가 결과 확인

### 4. 이메일 관리 (신규 기능!)
**이메일 관리** 탭에서 지원자와의 커뮤니케이션을 관리합니다:
- **발송 통계**: 발송 완료, 대기, 실패 건수 실시간 모니터링
- **새 이메일 발송**: 수신자 정보, 이메일 유형, 제목, 본문 작성
- **템플릿 자동 적용**: 이메일 유형 선택 시 자동으로 제목/본문 템플릿 입력
  - 지원서 접수 확인
  - 면접 일정 안내
  - 최종 합격 통보
  - 불합격 안내
- **발송 내역 조회**: 상태별, 유형별 필터링
- **재발송 기능**: 실패한 이메일 재전송

### 5. 통계 분석
**통계 분석** 탭에서 다음 정보를 확인할 수 있습니다:
- **채용 퍼널**: 단계별 지원자 수 시각화 (Bar Chart)
- **채용공고별 통계**: 지원자 수, 평균 매칭점수, 제안/불합격 현황
- **데이터 기반 인사이트**: 채용 효율성 분석

### 6. AI 매칭 기능
시스템은 자동으로 지원자와 채용공고의 적합도를 분석합니다:
- **학력 매칭**: 박사 +15점, 석사 +10점, 학사 +5점
- **경력 매칭**: 5년 이상 +15점, 3년 이상 +10점, 1년 이상 +5점
- **기술 스택 매칭**: 핵심 기술 보유 시 기술당 +3점 (최대 15점)
- **키워드 매칭**: 채용공고 요구사항 키워드 매칭 시 키워드당 +2점
- **최대 100점 만점**, 실시간 점수 계산

## 📝 개발 가이드

### 로컬 개발 환경 시작
```bash
# 의존성 설치 (최초 1회)
npm install

# 데이터베이스 마이그레이션
npm run db:migrate:local

# 샘플 데이터 삽입
npm run db:seed

# 빌드
npm run build

# 개발 서버 시작 (PM2)
pm2 start ecosystem.config.cjs

# 서버 상태 확인
pm2 list

# 로그 확인
pm2 logs --nostream
```

### 데이터베이스 관리
```bash
# 로컬 데이터베이스 초기화
npm run db:reset

# SQL 직접 실행
npm run db:console:local
```

### Git 관리
```bash
# 상태 확인
npm run git:status

# 커밋
npm run git:commit "메시지"

# 로그 확인
npm run git:log
```
